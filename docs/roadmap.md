# Roadmap

Our vision for Pridge: the most private, user-friendly cross-chain bridge to Solana.

## Completed

### Phase 1: Core MVP
- [x] Disposable keypair generation
- [x] Claim link system (URL fragment privacy)
- [x] Multi-chain support (7 EVM chains)
- [x] Embedded bridge UI (no external redirects)
- [x] Automatic wSOL unwrapping
- [x] Fee sponsoring via Phantom
- [x] Real-time balance detection
- [x] Clean, branded UI

---

## In Progress

### Phase 2: Enhanced Privacy
- [ ] **Light Protocol Integration** - Shielded transactions for amount privacy
- [ ] **Multiple Claim Outputs** - Split funds across multiple destinations
- [x] **Time-Locked Claims** - Set expiration on claim links

### Phase 3: Token Support
- [x] **USDC Bridging** - Stable coin support
- [x] **USDT Bridging** - Tether support
- [ ] **Custom Tokens** - Bridge any supported ERC-20

---

## Planned

### Phase 4: Advanced Features
- [ ] **Stealth Addresses** - Reusable private receiving addresses
- [ ] **Batch Operations** - Multiple bridges in one transaction
- [ ] **Mobile App** - Native iOS/Android apps
- [ ] **Hardware Wallet** - Ledger/Trezor support for claiming

### Phase 5: Ecosystem
- [ ] **SDK Release** - Integrate Pridge into your dApp
- [ ] **API Access** - Programmatic bridging
- [ ] **Multi-Destination** - Bridge to other chains (not just Solana)
- [ ] **DAO Governance** - Community-driven development

---

## Long-Term Vision

### Privacy Infrastructure
- Full end-to-end encryption
- Zero-knowledge proofs for verification
- Decentralized relayer network

### Cross-Chain Expansion
- Solana as source (not just destination)
- Bitcoin integration
- Cosmos ecosystem support
